>   **구글 해킹=도킹(Goodle Dorking)**

구글 검색 엔진의 강력한 필터링 기능을 활용하여 일반적인 검색으로는 찾기 힘든 민감한 정보나 시스템의 취약점을 찾아내는 기법
예를 들어 숨겨진 관리자 페이지, 비밀번호 파일, 유출된 인증 키를 찾아내는 강력한 도구

```

- site: 특정 사이트(도메인)의 결과만 한정하여 검색
    ㄴ site: *.target.system.com → 사이트페이지뿐만아니라 하위도메인들의 페이지까지 (삭제되지않은 관리되고있지않은페이지들)
- inurl: URL 주소 안에 특정 키워드가 포함된 페이지를 검색
    ㄴ site:*.normaltic.com inurl:admin → 하위도메인 + 관리자페이지
- intitle: 웹 페이지의 제목(Title) 부분에 특정 키워드가 있는 페이지를 찾음
    ㄴ intitle: index of  -> 디렉토리 리스팅 취약점 
- link: 특정 링크를 포함하고 있는 페이지를 식별
- filetype: xlsx, pdf, txt 등 특정 확장자를 가진 파일만 골라냄
```

>   **서브도메인 탐색**

TLS인증서를 통한 SAN(Subject Alternative Name, 하나의 인증서 안에 포함된 모든 도메인 목록) 파악
전 세계 인증기관(CA)은 발급한 TLS인증서를 CT로그에 공개해야하기에 "crt.sh" 검색 가능함

crt.sh에 남는이유는 공인 CA(Public CA)가 인증서를 발급했기 떄문에

```

https://crt.sh/?q=%25.bcto.me

(1)
curl -s "https://crt.sh/?q=%25.bcto.me&output=json" \
| jq -r '
  .[]
  | select(.not_before != null)
  | "\(.name_value) | \(.not_before) | \(.not_after)"
' \
| sort -u

(2)
curl -s "https://crt.sh/?q=%25.bcto.me&output=json" | jq
```
