PE파일의 악성 및 비정상 여부 식별 전략 (분석에 앞서)

1. 섹션의 엔트로피 분석
- 패킹 되었거나 암호화된 영역이 있는 실행 파일
- 각 게션마다 엔트로피 7이상인 경우 패킹, 인코딩 및 암호화 등 빈도 높음

2. 엔트리 포인트 분석
- 일반적으로 PE파일 경우 엔트리포인트 첫번째 섹션 속하며, ".text, .code" Section
- 엔트리포인트가 첫번째 섹션(".text, .code") 속하지 않는 경우 악성일 경우 높음

Entry Point(프로그램이 메모리에 적재된 이후에 가장 먼저 실행될 코드 위치)


3. CRC Checksum검사
- CRC(Cyclic Redundancy check)로서 실행파일 일정비트 별 체크섬 값 계산
- 보통은 전송되는데이터의 손상여부 확인하는 기술 중 하나이나, 시그니처 분석을 우회하는 Polymorphic 악성코드 판별 방법 중 하나

4. 버전 및 설명 정보 분석
- ".rsrc" Section - 관리를 위해 리소스 섹션 (파일의 버전,벤더,설명정보) -> 누락되거나 제대로 담겨져 있지 않은 경우

> sigcheck * 



5. 임포트한 API 목록 분석
- API를 임포트하는 실행 파일은 악성일 가능성 증가
VirtualAllocEx()
WriteProcessMemory()
CreateRemoteThread()
InternetOpenUrl()
InternetReadFile()

6. API 해시 사용여부 분석
- API 해시를 이용하여 동적 바인딩을 시도하는 실행 파일 (PE파일 안쪽의 특정 패턴 매핑 사전방식조사)

7. 실행파일 포함 여부
- 내부에 또 다른 실행 파일 포함

8. 배치 스크립트 포함 여부
- 내부에 배치 스크립트를 포함하고 있는 경우(Command 명령어)

9. 서명정보 분석
- 정상파일은 마이크로소프트 서명정책에 따라 개발자에 의해 생성된 서명정보가 임베드 되어 있음
(서명정보 파싱)

> sigcheck * 
명령어를 통한 Verified : Signed, Unsigned 통해서도 비정상파일 판별

프로세스 인젝션(다른프로세스의 가상 메모리 영역에 데이터를 주입) 개요 및 구현을 위한 API 사용 예

- 타겟 프로세스에 코드를 주입하고 실행시키는 기술 (디펜스 우회와 권한 상승을 위해)
- 기술의 핵심은 메모리 할당 및 코드 주입 방법, 코드 실행 방법이며 다양한 임플리멘테이션이 존재
- 프로세스 인젝션 탐지 및 mitigation 기술로는 CFG(control Flow Guard), CIG(Code Integrity Guard), API 모니터링, VAD Tree 조사 등이 있음

프로세스 인젝션 구현을 위한 API 사용 예
A process 실행하는데 -> OpenProcess -> VirtualAlloxEx (가상메모리 주소할당위한)
 -> WriteProcessMemory(메모리공간에 데이터를씀) ->CreateRemoteThread


실습
IDA에서 프로세스 인젝션 구현API들을 확인하고 주소확인 -> x32dbg에서 브레이크걸어서 분석
C:/Windows/system32에 위치할 것으로 예상되는 rdshost.dll이란 악성dll파일명을 WriteProcessMemory함수를 이용해 삽입

악성PE파일의 리소스 섹션(.rsrc/이미지,비디오,아이콘,문자열등등)과 분석전략  
->실행파일, 공격에 사용할 수 있는 스크립트, 셸파일이 담길 수 있음
why. 보안솔루션의 탐지우회를 위해 PE파일의 리소스섹션에 악성스크립트나 실행파일을 임베드

리소스섹션에 악성 데이터를 임베드 하는 악성코드가 보이는 징후
- 일반적으로 사용되지 않은 리소스 이름이 관찰됨
- 스크립트(VB, 파워쉘, 쉘 스크립트)가 관찰됨
- 엔트로피(공백이없고촘촘해서, 패턴이 보이지않아 무작위성 높음) 값이 높은 바이너리 스트림을 담은 리소스가 관찰됨
- MZ(4D, 5A) 시그니처를 포함한 리소스가 관찰됨
- 리소스 섹션에 임베드된 데이터를 접근할 때 사용하는 API들이 관찰됨
cf. 리소스섹션에 임베드된 데이터를 접근할 때 사용되는 API
   (FindResource, SizeofResource, LoadResource, LockResource)
