**앱 디렉토리 체계(샌드박스 구조)**

iOS앱은 설치 시 보안을 위해 격리된 공간(샌드박스)에 배치되며, 크게 세 가지 컨테이너로 나뉨
-Bundle Container : 앱의 실행 바이너리(Executable), 리소스, 그리고 Info.plist가 위치하는 읽기 전용 영역임. 번들 ID로 경로가 생성됨
  ㄴ /var/containers/Bundle/Application/<'UUID>
-Data Container : 앱이 실행되면서 생성하거나 저장하는 데이터가 위치함
  ㄴ /var/mobile/containers/Date/Application/<'UUID>
  -Documents: 사용자가 생성한 콘텐츠나 오프라인 데이터 저장. iTunes 백업시 포함되므로 민감정보 저장시 주의필요
  -Library : 앱 설정 및 캐시 데이터 저장. 특히 Preferences폴더에는 사용자 설정 정보 저장
  -Tmp : 실행 중 임시 파일을 저장. 앱 종료 시 OS에 의해 자동삭제됨
-iCloud Container : iCloud 동기화를 사용하는 앱을 위한 데이터 저장 공간
  ㄴ /var/mobile/Library/Mobile Documents/<'TeamID><'BundleID>




IOS앱은 App Delegate라는 객체가 앱의 상태 변화(실행, 백그라운드 등)를 감지하고 처리함.

생명주기
Not Running -> Inactive -> Active -> Background ->Suspended 순서로 상태 변화

주요 델리데이트 메소드
-didFinishLaunchingWithOptions : 앱이 실행되고 화면에 보이기 직전에 호출됨
-didBecomeActive : 앱이 활성화되어 사용자 이벤트를 받을 수 있는 상태가 된 직후 호출됨

Inactive상태 : 앱이 화면에는 보이지만 알림이나 시스템 팝업에 가려져 터치가 불가능한 상태
